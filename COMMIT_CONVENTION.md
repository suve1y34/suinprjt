# Commit Message Convention

## 🔹 기본 원칙
- 형식: `[type] 간단한 제목`
- 제목은 명령형
- 필요시 본문에 상세 설명 추가

---

## 🔹 타입(type) 정의

| 타입 | 설명 |
|:---|:---|
| **feat** | 새로운 기능 추가 |
| **fix** | 버그 수정 |
| **docs** | 문서 수정 (README, 설명서 등) |
| **style** | 코드 포맷팅/형식 정리 (기능 변화 없음) |
| **refactor** | 코드 리팩토링 (기능 변화 없음) |
| **test** | 테스트 코드 추가/수정 |
| **chore** | 빌드 설정, 패키지 수정 등 |
